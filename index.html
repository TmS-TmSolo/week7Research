What are the differences between var, let, and const? https://javascripttoday.com/blog/var-vs-let-vs-const/ The main differential between the 3 are the type of scopes they have. Var is scoped outside of the block or globally as let and const are scoped inside of the block or block scoped. The difference in scope can lead to problems and/or limitations on both ends. Var being function scoped means it can be used in the entirety of the function scope, so we must be careful how we are using it. Let was created to somewhat replace Var as it is only accessible in the block it is in, allowing it to be changed later without disruption or error. Let and Const are similar in scope but differing in alteration; let can be changed as const stays constant throughout. Let was brought in to replace var, so of course it should be used more frequently and const has it’s very reason to be used with consistency. What are the differences between callbacks and promises? https://www.geeksforgeeks.org/promise-vs-callback-in-javascript/ https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Using_promises Promises are objects representing the answer of two or more events returning true or false, correct or wrong. Callbacks are a function that passes an argument into another function to return an action, somewhat answering for the response of the function’s happenings. The way the two are chained and the way the two are read are two main differences. Promises are forward in readability, purely representing the completion or failure of an asynchronous event. Callbacks are used inside of a function to callback to get a result upon the completion and are attached to promises. Promises are telling the computer that this will happen as callbacks are saying when this happens do this covering their basis.
